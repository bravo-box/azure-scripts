# Scripts

Collection of PowerShell and Bash helpers for Azure services. THe readme outlines the file and the function it performs. These were built to help simplify common tasks, but also to create some repeatability which can be adjusted and re-used in "production" settings.

## Script Inventory

### PowerShell

| Script | Purpose | Notes |
|---|---|---|
| `entra/entra_sync_encrypt.ps1` | Encrypts Entra app registration credentials (tenant ID, client ID, client secret) with DPAPI and stores them on disk. | Must run on the same machine that will run the sync script. Uses `LocalMachine` scope encryption and writes files under `C:\Secure\AzureApp`. |
| `entra/entra_sync.ps1` | Syncs users from a specified Entra security group into on-prem AD (creates missing AD users). | Reads encrypted credentials generated by `entra_sync_encrypt.ps1`. Writes text/json logs and Event Log entries. Requires AD and Microsoft Graph PowerShell modules and proper app permissions. |
| `azure-local/node-prep/node-prep.ps1` | Prepares an Azure Local node by creating a working folder, downloading bootstrap script/driver package, and showing network driver providers. | Default driver URL is HPE-specific and should be changed for non-HPE hardware. |
| `azure-local/node-prep/azurelocal-node-add.ps1` | Initializes Azure Local Arc onboarding context (login, subscription/tenant capture, and Arc initialization call). | Interactive script. Prompts for environment and resource group. Expects required resource providers to already be registered. This removes the need to copy and paste subscription and tenant details. |
| `azure-local/create-logical-network/create-azl-lnet.ps1` | Creates or updates Azure Local logical networks from JSON config using Azure CLI `stack-hci-vm network lnet` commands. | Interactive when `subscriptionId`, `resourceGroup`, `customLocationId`, `location`, or `vmSwitchName` are missing. Supports `-WhatIf` preview and `-UpdateIfExists` updates. |
| `azure-local/create-vmimages/Deploy-AzLocalVMImage.ps1` | Deploys Azure Local VM gallery images from storage-backed VHD/VHDX files using an ARM template and runtime-generated parameters. | Supports interactive custom location selection, storage auto-discovery, SAS-based secure `imagePath`, and multi-image selection from a container. |

### Bash

| Script | Purpose | Notes |
|---|---|---|
| `azure-local/arc-enabled-kubernetes/k8s_proxy.sh` | Starts an Azure Arc-enabled Kubernetes proxy session and supports interactive cluster selection. | Checks for `az`, `kubectl`, `jq`, and installs `connectedk8s` extension if missing. Logs to a timestamped file. If there are multiple clusters in the subscription you will be prompted to select the cluster you want to work on. |
| `azure-local/arc-enabled-kubernetes/generate-service-token.sh` | Creates a Kubernetes service token tied to the signed-in Azure user object and copies it to clipboard. | Requires valid Azure login and active `kubectl` context. Uses `pbcopy` (macOS) to copy the service token to memory. Service token is also printed to screen. |
| `azure-local/arc-enabled-kubernetes/sql-on-aks.sh` | Deploys SQL Server on an Arc-enabled AKS cluster (namespace, secret, PVC/statefulset/service, load balancer). | Interactive script. Prompts for SA password and LB IP range. Requires existing storage class named `default`. |
| `azure-local/image-prep/generalize-rhel.sh` | Prepares an Azure-created RHEL VM to boot outside Azure by switching cloud-init datasource and optionally deprovisioning host identity. | Supports safe defaults, file backup, optional waagent retention, and `--deprovision` mode for image capture workflows. See `azure-local/image-prep/readme.md` for detailed guidance. |

## Prerequisites

### PowerShell scripts

- PowerShell 7+ recommended
- Azure PowerShell modules (`Az`)
- For Entra sync: `Microsoft.Graph` module and `ActiveDirectory` module
- For logical network creation: Azure CLI (`az`) with `stack-hci-vm` extension and authenticated session (`az login`)
- Permissions to create/read event logs and write to local paths used by scripts

### Bash scripts

- `bash`
- Azure CLI (`az`)
- `kubectl`
- `jq` (required by `k8s_proxy.sh`)
- Access to Arc-enabled Kubernetes clusters where applicable

## Usage Examples

```bash
# Start Arc Kubernetes proxy (interactive cluster selection)
./azure-local/arc-enabled-kubernetes/k8s_proxy.sh

# Start Arc Kubernetes proxy for a specific cluster
./azure-local/arc-enabled-kubernetes/k8s_proxy.sh my-cluster-name

# Generate a service token and copy to clipboard
./azure-local/arc-enabled-kubernetes/generate-service-token.sh

# Deploy SQL Server on AKS/Arc
./azure-local/arc-enabled-kubernetes/sql-on-aks.sh
```

```powershell
# 1) One-time setup: encrypt Entra app credentials on the sync host
pwsh .\entra\entra_sync_encrypt.ps1

# 2) Run Entra to AD sync
pwsh .\entra\entra_sync.ps1

# Prepare Azure Local node
pwsh .\azure-local\node-prep\node-prep.ps1

# Run Azure Local Arc initialization flow
pwsh .\azure-local\node-prep\azurelocal-node-add.ps1

# Preview logical network create/update operations
pwsh .\azure-local\create-logical-network\create-azl-lnet.ps1 -WhatIf

# Apply logical network operations from config
pwsh .\azure-local\create-logical-network\create-azl-lnet.ps1

# Update existing logical networks if they already exist
pwsh .\azure-local\create-logical-network\create-azl-lnet.ps1 -UpdateIfExists

# Deploy Azure Local VM image(s)
pwsh .\azure-local\create-vmimages\Deploy-AzLocalVMImage.ps1
```

## Configuration Points

- Update placeholder values before running:
	- `entra/entra_sync_encrypt.ps1`: `YOUR-TENANT-ID`, `YOUR-CLIENT-ID`, `YOUR-CLIENT-SECRET`
	- `entra/entra_sync.ps1`: `AD_Security_Group_Name`, `OU=...`, default password variable, secure file paths
	- `azure-local/create-logical-network/create-azl-lnet.config.json`: set `subscriptionId`, `resourceGroup`, `customLocationId`, and `logicalNetworks` values as needed (or leave selected fields blank for interactive prompts)
- Review environment assumptions:
	- `azure-local/node-prep/node-prep.ps1` uses an HPE driver URL by default
	- `azure-local/create-logical-network/create-azl-lnet.ps1` requires valid Azure context and uses existing custom locations/vSwitch discovery when config fields are omitted
	- `azure-local/arc-enabled-kubernetes/sql-on-aks.sh` expects a storage class named `default`

## Safety Notes

- Do not commit real tenant IDs, client secrets, passwords, or generated tokens.
- Treat generated service tokens as sensitive credentials.
- Test scripts in a non-production environment first.
