# Scripts

Collection of PowerShell and Bash helpers for Azure services. THe readme outlines the file and the function it performs. These were built to help simplify common tasks, but also to create some repeatability which can be adjusted and re-used in "production" settings.

## Script Inventory

### PowerShell

| Script | Purpose | Notes |
|---|---|---|
| `powershell/entra_sync_encrypt.ps1` | Encrypts Entra app registration credentials (tenant ID, client ID, client secret) with DPAPI and stores them on disk. | Must run on the same machine that will run the sync script. Uses `LocalMachine` scope encryption and writes files under `C:\Secure\AzureApp`. |
| `powershell/entra_sync.ps1` | Syncs users from a specified Entra security group into on-prem AD (creates missing AD users). | Reads encrypted credentials generated by `entra_sync_encrypt.ps1`. Writes text/json logs and Event Log entries. Requires AD and Microsoft Graph PowerShell modules and proper app permissions. |
| `powershell/azure-local/node-prep.ps1` | Prepares an Azure Local node by creating a working folder, downloading bootstrap script/driver package, and showing network driver providers. | Default driver URL is HPE-specific and should be changed for non-HPE hardware. |
| `powershell/azure-local/azurelocal-node-add.ps1` | Initializes Azure Local Arc onboarding context (login, subscription/tenant capture, and Arc initialization call). | Interactive script. Prompts for environment and resource group. Expects required resource providers to already be registered. This removes the need to copy and paste subscription and tenant details. |

### Bash

| Script | Purpose | Notes |
|---|---|---|
| `bash/azure-local/k8s_proxy.sh` | Starts an Azure Arc-enabled Kubernetes proxy session and supports interactive cluster selection. | Checks for `az`, `kubectl`, `jq`, and installs `connectedk8s` extension if missing. Logs to a timestamped file. If there are multiple clusters in the subscription you will be prompted to select the cluster you want to work on. |
| `bash/azure-local/generate-service-token.sh` | Creates a Kubernetes service token tied to the signed-in Azure user object and copies it to clipboard. | Requires valid Azure login and active `kubectl` context. Uses `pbcopy` (macOS) to copy the service token to memory. Service token is also printed to screen. |
| `bash/azure-local/sql-on-aks.sh` | Deploys SQL Server on an Arc-enabled AKS cluster (namespace, secret, PVC/statefulset/service, load balancer). | Interactive script. Prompts for SA password and LB IP range. Requires existing storage class named `default`. |

## Prerequisites

### PowerShell scripts

- PowerShell 7+ recommended
- Azure PowerShell modules (`Az`)
- For Entra sync: `Microsoft.Graph` module and `ActiveDirectory` module
- Permissions to create/read event logs and write to local paths used by scripts

### Bash scripts

- `bash`
- Azure CLI (`az`)
- `kubectl`
- `jq` (required by `k8s_proxy.sh`)
- Access to Arc-enabled Kubernetes clusters where applicable

## Usage Examples

```bash
# Start Arc Kubernetes proxy (interactive cluster selection)
./bash/azure-local/k8s_proxy.sh

# Start Arc Kubernetes proxy for a specific cluster
./bash/azure-local/k8s_proxy.sh my-cluster-name

# Generate a service token and copy to clipboard
./bash/azure-local/generate-service-token.sh

# Deploy SQL Server on AKS/Arc
./bash/azure-local/sql-on-aks.sh
```

```powershell
# 1) One-time setup: encrypt Entra app credentials on the sync host
pwsh .\powershell\entra_sync_encrypt.ps1

# 2) Run Entra to AD sync
pwsh .\powershell\entra_sync.ps1

# Prepare Azure Local node
pwsh .\powershell\azure-local\node-prep.ps1

# Run Azure Local Arc initialization flow
pwsh .\powershell\azure-local\azurelocal-node-add.ps1
```

## Configuration Points

- Update placeholder values before running:
	- `powershell/entra_sync_encrypt.ps1`: `YOUR-TENANT-ID`, `YOUR-CLIENT-ID`, `YOUR-CLIENT-SECRET`
	- `powershell/entra_sync.ps1`: `AD_Security_Group_Name`, `OU=...`, default password variable, secure file paths
- Review environment assumptions:
	- `powershell/azure-local/node-prep.ps1` uses an HPE driver URL by default
	- `bash/azure-local/sql-on-aks.sh` expects a storage class named `default`

## Safety Notes

- Do not commit real tenant IDs, client secrets, passwords, or generated tokens.
- Treat generated service tokens as sensitive credentials.
- Test scripts in a non-production environment first.
